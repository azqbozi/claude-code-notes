
# 安装与启动
## 一、核心概念
1. 工具定位：终端中的编码辅助工具，支持写代码、调bug、自动化任务
2. 安装核心原则：优先选 Native Install 方式（自动更新，无需手动升级）
3. 适用场景：从零搭建工具环境，为后续开发操作打基础

## 二、完整操作步骤（含命令+目的）
| 步骤 | 操作+命令 | 核心目的 | 对应网页知识点 |
|------|-----------|----------|----------------|
| 1 | 右键点击「Windows PowerShell」→ 选择「以管理员身份运行」 | 获取足够系统权限，避免安装报错 | 安装前置准备 |
| 2 | 输入命令并回车：<br>`irm https://claude.ai/install.ps1 | iex` | 执行 Windows PowerShell 专属 Native 安装，自动下载最新版 | Windows PowerShell 对应的 Native Install 命令 |
| 3 | 输入命令并回车：<br>`claude --version` | 验证安装是否成功（成功会显示版本号） | 安装结果校验 |
| 4 | 输入命令并回车（替换为自己的项目路径）：<br>`cd C:\Users\你的用户名\Documents\目标项目文件夹` | 让 Claude Code 识别当前项目代码结构 | 启动前需进入项目目录 |
| 5 | 输入命令并回车：<br>`claude` | 启动工具，触发首次登录流程 | 工具启动命令 |
| 6 | 按照终端提示，登录 Claude.ai 账户 | 完成身份验证，解锁全部功能 | 首次使用登录要求 |

## 三、常见问题与解决方案
1. 权限被拒绝报错：
   - 执行命令：`Set-ExecutionPolicy RemoteSigned -Scope CurrentUser`
   - 按提示输入 `Y` 确认，解除脚本执行限制后重新安装
2. 项目路径输入麻烦：
   - 打开目标项目文件夹 → 点击地址栏 → 输入 `powershell` 回车，直接在当前目录打开终端

## 四、关键要点总结
1. 必须使用网页推荐的 Native Install 命令，确保自动更新功能
2. 启动前务必进入项目目录，否则工具无法识别项目结构
3. 管理员身份运行 PowerShell 是避免安装失败的关键前提
